# Experiment 6: `ex6_grid_source_localization.py` — Source Localization on Grid

## Motivation

Experiment 5 showed that source localization via the meta-level backward model
fails when the target distributions are near-uniform — the inverse problem is
ill-conditioned because many distinct sources map to nearly identical observations.

Experiment 6 demonstrates source localization on a well-posed problem: recovering
the origin of a diffused bump on a 2D grid. Different source locations produce
spatially distinct diffusion patterns, making the inverse problem well-conditioned.
This is the discrete analogue of solving the heat equation backward.

## Setup

- **Graph**: 5x5 grid graph (25 nodes), unweighted
- **Source distributions (Pi_0)**: delta-like peaks at various grid positions.
  For each source, pick a node (r, c), place mass 0.8 there, distribute 0.2
  uniformly among the rest, add small noise (std=0.02), clip, renormalize.
  Generate 50 source distributions, sampling peak locations uniformly from
  all 25 grid positions.
- **Target distributions (Pi_1)**: the DIFFUSED versions of the sources.
  For each source distribution mu_source:
    1. Compute the exact forward flow from mu_source toward the uniform
       distribution (using the graph-level solver with the uniform as target).
    2. Evaluate the marginal distribution at t=0.5 (halfway diffused).
       This gives a "bump" that is spread but still clearly localized.
    3. This intermediate distribution is the target.
  
  The choice of t=0.5 is a balance: enough diffusion to be nontrivial, but
  the bump should still be clearly centered near the source node. Verify
  visually that bumps from different source locations are distinguishable.

- **Backward model task**: given a diffused bump (Pi_1), recover the peaked
  source (Pi_0). This is now well-posed — different bumps look different.

## Important: Generating the Diffusion Targets

The targets should be generated by actual diffusion, not by some ad hoc
smoothing. Use the graph-level exact solver:

```python
for each source distribution mu_source:
    # Transport from mu_source toward uniform
    mu_uniform = np.ones(N) / N
    cost = compute_cost_matrix(graph)
    coupling = compute_ot_coupling(mu_source, mu_uniform, cost)
    
    # Get the distribution at t=0.5
    mu_diffused = marginal_distribution(graph, coupling, t=0.5)
    
    # This is our "observation" / target for the backward model
```

Alternatively, since we just want diffusion (not OT toward uniform), we can
directly evolve under the reference random walk:

```python
    # Simpler: just apply the heat kernel
    # p(t) = p(0) @ expm(t * R) where R is the graph rate matrix
    from scipy.linalg import expm
    diffusion_time = 2.0  # tune this — controls how spread the bump is
    mu_diffused = mu_source @ expm(diffusion_time * graph.R)
```

The second approach (heat kernel) is cleaner and more direct. The diffusion_time
parameter controls difficulty:
- diffusion_time = 0.5: tight bump, easy localization
- diffusion_time = 2.0: moderate spread, good for proof of concept
- diffusion_time = 5.0: very spread, harder

Use diffusion_time = 2.0 as default. Verify visually that the bumps are
distinguishable but nontrivially spread.

## Training the Backward Model

1. Generate 50 source-target pairs (peaked, diffused) as described above.
2. Compute cost matrix c(i,j) on the grid graph.
3. For each pair (mu_diffused, mu_peaked):
   Compute graph-level OT coupling transporting mu_diffused to mu_peaked.
4. Compute meta-cost matrix W(mu_diffused_s, mu_peaked_t) for all pairs.
5. Solve meta-OT coupling.
6. Pre-generate 5000 training samples (mu_tau, tau, R_target).
7. Train GNNRateMatrixPredictor:
   - n_nodes = 25
   - hidden_dim = 64 (same as Experiments 3/4)
   - n_layers = 4 (same as Experiments 3/4)
   - Train for 500 epochs (same as Experiments 3/4)
   - Learning rate 1e-3 with Adam (same as Experiments 3/4)
   If loss is still decreasing at 500, extend to 1000.
8. Save checkpoint as checkpoints/meta_model_grid_backward_gnn.pt

## Evaluation

### Test Cases

Generate 15 held-out test cases:
- 15 peaked distributions at random grid positions (not in training set, but
  positions may overlap — that's fine, the distribution values differ due to noise)
- For each, compute the diffused observation using the same heat kernel
- Add observation noise at three levels: sigma = 0.0, 0.02, 0.05

### Source Recovery

For each test case:
1. Start from the diffused observation mu_diffused (+ noise)
2. Run backward model forward: integrate dp/dt = p @ R_backward(p, t)
   from t=0 to t=1 using Euler (n_steps=200)
3. Output at t=1 is the recovered source mu_recovered
4. Compare to true peaked source mu_source

### Metrics

- **TV distance**: between recovered and true source
- **Peak recovery**: does argmax(mu_recovered) match argmax(mu_source)?
- **Localization error**: grid distance between recovered peak and true peak
  d_grid = |r_recovered - r_true| + |c_recovered - c_true| (Manhattan distance)
  This is a softer metric — being off by one cell is much better than being
  on the opposite corner.

## Expected Outcome

This should work substantially better than Experiment 5 because:
1. Observations (diffused bumps) are spatially distinct for different sources
2. The GNN can exploit spatial structure on the grid
3. The backward map is approximately one-to-one

Expected performance at sigma=0:
- Peak recovery: >60%
- Mean localization error: <2 (within 2 grid steps of true source)
- TV distance: <0.3

At moderate noise, graceful degradation. Even when the exact peak is missed,
the recovered peak should be spatially close to the true source.
